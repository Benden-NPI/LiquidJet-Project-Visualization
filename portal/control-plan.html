Holistic Layer — Qualify 強制 Exit Criteria

    Self-audit — 每週 5 分鐘逐 station

    1. 目前 milestone = ？下一個 milestone exit criteria 全部列得出來嗎？
    2. Move-in/Setup/Tuning/Qualify 該長出的元件，有沒有「不確定」？
    3. 該 station PFMEA 高 RPN cause，是否都對到 control plan 一行？
    4. 每行 control 是否都對到 FMEA cause 且標明 milestone 歸屬？
    5. tuningCriteria / qualifyCriteria 字串完整定義在 scope/tuning-qualify-criteria.md 找得到？
    6. 三 owner（SurveyTool / EE / NPI）上週是否各看過一次？
    7. 上週 NG / 異常是否都進到 CAPA 或 SPC 看板？

    從 Yield 同步 46 站 (Power Automate)

    使用與 Yield · Tool Gantt 相同的 Power Automate HTTP GET webhook，讀取 Control Plan Excel 的 Process Flow sheet，把 station 清單覆寫到本機。

    URL 與 Yield 共用 localStorage key（tool_gantt_sharepoint_webhook_url）— 若你已在 Yield Settings 設過，此處會自動帶入。URL 含 sig= token，只存本機瀏覽器，不會 commit。

    回到內建 SSOT

    清除本機 override，改用 build-time 嵌入的 scope/stations.json。同步過的資料會被丟棄（你隨時可再同步一次）。

    除錯

    同步過程的詳細 log 可在瀏覽器 DevTools Console 查看（[CP-Sync] 前綴）。
